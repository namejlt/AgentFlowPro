## 1. 系统概述

### 1.1 什么是 AgentFlow Pro

AgentFlow Pro 是一个**智能体驱动的工作流引擎**，核心思想是：

```
配置 LLM模型 + 外部数据源 + 智能体
        ↓
    搭建成工作流（DAG 有向无环图）
        ↓
    执行工作流 → 生成专业分析报告
```

系统将复杂的分析任务拆解为多个步骤，每个步骤由一个专门的**智能体（Agent）**负责，智能体可以绑定**LLM 模型**和**外部数据源**，按照工作流编排顺序执行，最终汇总输出一份完整的 Markdown 报告。

### 1.2 核心概念

| 概念 | 说明 |
|------|------|
| **LLM 模型** | 大语言模型驱动，支持 OpenAI、DeepSeek、通义千问等 |
| **数据源** | 外部数据接入，支持 HTTP API、手动输入、文件上传 |
| **智能体** | 绑定模型+数据源的专业分析角色（如股票分析师） |
| **工作流** | 智能体的执行编排，DAG 图结构 |
| **任务** | 工作流的一次执行实例 |
| **报告** | 任务执行完成后生成的最终分析文档 |

### 1.3 系统架构

```
用户输入（股票代码、分析参数等）
        ↓
  工作流引擎（DAG 调度器）
        ↓
  ┌──────┐  ┌──────┐  ┌──────────┐
  │智能体A│→│智能体B│→│ 辩论/验证 │→ 汇总
  │+数据源│  │+数据源│  │ +风险评审 │
  └──────┘  └──────┘  └──────────┘
        ↓
  最终报告（Markdown）
```

## 2. 快速开始

### 2.1 5 分钟快速上手

**步骤 1：配置 LLM 模型**

进入「模型管理」→ 点击「添加模型」，填写：
- **名称**：如 `DeepSeek-V3`
- **厂商**：如 `deepseek`
- **接口地址**：如 `https://api.deepseek.com/v1/chat/completions`
- **模型名**：如 `deepseek-chat`
- **API Key**：你的 API 密钥
- **启用流式输出**：建议开启（实时显示输出内容）

**步骤 2：创建智能体**

进入「智能体管理」→ 点击「创建智能体」，填写：
- **名称**：如 `股票分析师`
- **角色描述**：如 `专业股票分析师，擅长技术面分析`
- **System Prompt**：编写专业的分析指令（详见第 5 章）
- **绑定模型**：选择刚创建的 LLM 模型
- **绑定数据源**：选择一个或多个数据源

**步骤 3：创建工作流**

进入「工作流管理」→ 点击「创建工作流」，拖拽节点：
```
[开始] → [智能体执行] → [辩论] → [风险评审] → [汇总] → [结束]
```

**步骤 4：运行工作流**

在工作流详情页点击「运行」，输入参数（如股票代码 `sh600519`），点击确定。

**步骤 5：查看报告**

任务执行完成后，在「报告管理」中查看生成的 Markdown 报告，支持导出为 PDF/DOCX。

## 3. LLM 模型配置

### 3.1 支持的模型厂商

| 厂商 | 端点格式 | 模型示例 |
|------|---------|---------|
| **OpenAI** | `https://api.openai.com/v1/chat/completions` | `gpt-4o`, `gpt-4o-mini` |
| **DeepSeek** | `https://api.deepseek.com/v1/chat/completions` | `deepseek-chat`, `deepseek-reasoner` |
| **通义千问** | `https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions` | `qwen-max`, `qwen-plus` |
| **Claude** | `https://api.anthropic.com/v1/chat/completions` | `claude-3.5-sonnet` |
| **智谱** | `https://open.bigmodel.cn/api/paas/v4/chat/completions` | `glm-4-plus` |
| **Kimi** | `https://api.moonshot.cn/v1/chat/completions` | `moonshot-v1-auto` |

### 3.2 配置参数详解

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| **名称** | 自定义模型名称，方便识别 | `DeepSeek-V3` |
| **厂商** | 模型厂商标识 | `deepseek` |
| **接口地址** | OpenAI 兼容的 Chat Completion 端点 | 见上表 |
| **模型名** | 具体模型 ID | `deepseek-chat` |
| **API Key** | 从模型厂商获取的 API 密钥 | — |
| **温度** | 生成随机性 0.0~2.0，越低越确定 | `0.3~0.7` |
| **最大 Token** | 单次生成的最大 token 数 | `4096~8192` |
| **超时(ms)** | 单次请求超时毫秒 | `60000~300000` |
| **重试次数** | 失败后重试次数 | `2~3` |
| **流式输出** | 开启后实时显示生成内容 | ✅ 推荐开启 |

### 3.3 配置建议

- **分析类智能体**：温度 0.3~0.5，追求准确性和稳定性
- **创意类智能体**：温度 0.7~0.9，允许更多创造性
- **汇总类智能体**：温度 0.3，要求精确总结
- **超时设置**：如果智能体分析的数据量大，建议设到 180 秒以上
- **测试模型**：配置后点击「测试」按钮验证连通性

### 3.4 设置默认模型

在模型列表页，可以将某个模型设为默认。工作流执行时，如果智能体没有指定模型，会自动使用默认模型。

## 4. 外部数据源配置

### 4.1 数据源类型

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| **HTTP_GET** | GET 请求获取数据 | 查询行情、天气、公开 API |
| **HTTP_POST** | POST 请求提交参数获取数据 | 带复杂查询条件的 API |
| **手动输入 (MANUAL_INPUT)** | 用户在工作流运行时手动输入 | 私有数据、静态分析 |
| **文件上传 (FILE_UPLOAD)** | 上传本地文件进行分析 | CSV、JSON、TXT 分析 |

### 4.2 参数配置详解

#### HTTP_GET 数据源

```
┌─────────────────────────────────────────────────────────┐
│ 名称：股票实时行情                                        │
│ URL模板：https://qt.gtimg.cn/q={{stock_code}}            │
│ HTTP方法：GET                                            │
│ 参数：                                                   │
│   stock_code = {{stock_code}}  (来源：全局变量)            │
│ 响应提取路径：$                                           │
│ 缓存：TTL 60秒                                            │
└─────────────────────────────────────────────────────────┘
```

**URL 模板**：支持 `{{变量名}}` 占位符，执行时会被实际值替换：
- `{{stock_code}}` — 从工作流的**输入参数**中获取
- `{{industry}}`、`{{keyword}}` — 自定义变量

**响应提取路径**：使用 GJSON 语法提取 JSON 响应中的特定字段：
- `$` — 返回全部数据
- `$.data` — 提取 data 字段
- `$.articles.0.title` — 提取第一条文章标题
- `$.rates.CNY` — 提取嵌套的汇率值

#### 手动输入数据源

```
┌─────────────────────────────────────────────────────────┐
│ 名称：手动输入数据                                          │
│ 参数：                                                   │
│   content = ____________  (来源：运行时输入)               │
│   （用户在运行工作流时填写此内容）                            │
└─────────────────────────────────────────────────────────┘
```

参数来源类型：
- **global_var** — 从工作流输入参数中取值（如 `stock_code`）
- **fixed_value** — 固定值，所有任务使用相同参数（如 `scale=240`）
- **runtime_input** — 运行时由用户手动填写

### 4.3 内置数据源说明

系统预置了以下数据源：

| 数据源 | 类型 | 参数 | 说明 |
|--------|------|------|------|
| **股票实时行情** | HTTP_GET | `stock_code` | 腾讯财经，实时报价+PE/PB |
| **股票历史行情** | HTTP_GET | `stock_code` | 腾讯财经，日K线历史数据 |
| **股票基本面数据** | HTTP_GET | `stock_code` | 腾讯财经，基本面指标 |
| **股票财务指标** | HTTP_GET | `stock_code` | 腾讯财经，技术分析数据 |
| **沪深行情概览** | HTTP_GET | — | 大盘指数行情 |
| **年报财务数据** | HTTP_GET | `stock_code_full` | 东方财富，多年年报财务数据 |
| **汇率查询** | HTTP_GET | `base_currency` | 实时汇率（无需 Key） |
| **天气查询** | HTTP_GET | `lat`, `lon` | 天气预报（无需 Key） |
| **高考分数线查询** | MANUAL_INPUT | 省份、年份、批次 | 手动输入 |
| **高校专业信息** | MANUAL_INPUT | 学校、专业 | 手动输入 |
| **手动输入数据** | MANUAL_INPUT | 自定义内容 | 运行时填写 |

> **注意**：使用"年报财务数据"时，股票代码需使用大写格式，如 `SH600519`（而非 `sh600519`），因为东方财富 API 要求此格式。

### 4.4 创建自定义数据源

**示例：添加一个新闻搜索 API**

```yaml
名称: 新闻搜索
类型: HTTP_GET
URL模板: https://api.example.com/news?q={{keyword}}&pageSize=10
参数:
  - keyword: 搜索关键词 (来源: global_var)
响应提取路径: $.results
缓存: TTL 300秒
```

**示例：添加一个需要 API Key 的数据源**

配置方法：
1. URL 模板中正常使用参数
2. 认证方式选择 `api_key_header` 或 `bearer`
3. 在认证配置中填写 Header 名称和 API Key 值
4. API Key 会被加密存储

### 4.5 测试数据源

配置完成后，点击「测试」按钮：
1. 输入测试参数值（如 `sh600519`）
2. 系统会实际调用该数据源
3. 返回的数据会显示在预览窗口
4. 如果返回错误，根据错误信息调整配置

### 4.6 缓存策略

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| **none** | 不缓存，每次重新请求 | 实时数据、动态数据 |
| **ttl** | 指定过期时间（秒）内复用 | 行情数据（60s）、汇率（1h） |
| **fixed** | 固定缓存，不过期 | 年报数据（24h） |

## 5. 智能体配置

### 5.1 智能体概念

智能体（Agent）是一个**分析角色**，它绑定了一个 LLM 模型和一个或多个数据源，通过 System Prompt 定义其分析方向和输出格式。

```
智能体 = LLM 模型 + 数据源 + System Prompt
```

### 5.2 配置参数

| 参数 | 说明 | 示例值 |
|------|------|--------|
| **名称** | 智能体名称 | `股票分析师` |
| **角色描述** | 一句话描述职责 | `专业股票分析师，擅长技术面和基本面分析` |
| **头像图标** | 可视化标识 | `TrendCharts` |
| **System Prompt** | 系统提示词（核心） | 见下方模板 |
| **绑定模型** | 使用的 LLM 模型 | `DeepSeek-V3` |
| **绑定数据源** | 一个或多个数据源 | `股票实时行情`、`股票历史行情` |
| **输出格式** | 输出格式 | `markdown` / `plaintext` / `json` |
| **输出语言** | 输出语言 | `zh-CN` |
| **最大输出字符** | 输出长度限制 | `8000` |

### 5.3 System Prompt 编写指南

System Prompt 是智能体的灵魂，它定义了：
- 智能体的**角色身份**
- 分析的**维度和框架**
- 输出的**格式要求**
- 使用的**模板变量**

#### 模板变量

在 System Prompt 中可以引用以下变量（执行时自动替换）：

| 变量 | 说明 | 示例值 |
|------|------|--------|
| `{{datasource.result}}` | 绑定数据源返回的数据 | 行情数据、K线数据 |
| `{{stock_code}}` | 工作流输入参数 | `sh600519` |
| `{{current_date}}` | 当前日期（自动注入） | `2026-05-14` |
| `{{report_date}}` | 报告日期（中文格式） | `2026年05月14日` |
| `{{current_time}}` | 当前时间 | `15:01:32` |
| `{{agent:uuid:output}}` | 指定智能体的输出 | 上游分析结果 |

#### 智能体 Prompt 模板示例

**股票分析师：**

```
你是一位资深股票分析师，拥有 CFA 和 FRM 双证。
当前日期：{{report_date}}
分析标的：{{stock_code}}

请基于以下数据源提供的数据进行专业分析：
{{datasource.result}}

分析框架：
1. 技术面分析：K线形态、均线系统、成交量、MACD/KDJ指标
2. 基本面分析：PE/PB估值、营收增长、净利润、ROE
3. 市场情绪：资金流向、换手率、市场热度
4. 风险提示：波动率、政策风险、行业风险
5. 投资建议：短期/中期/长期策略

请用 Markdown 格式输出专业分析报告。
```

**行业研究员：**

```
你是一位行业研究总监，专注于产业链深度分析。
当前日期：{{report_date}}

数据来源：
{{datasource.result}}

分析框架：
1. 行业概况：市场规模、增长率、生命周期阶段
2. 竞争格局：CR5集中度、护城河分析、波特五力
3. 产业链：上游供应商议价能力、下游客户结构
4. 政策环境：监管政策、补贴政策、国际贸易影响
5. 未来趋势：技术变革、替代品威胁、新进入者

输出格式为 Markdown，要求逻辑清晰、数据支撑充分。
```

### 5.4 智能体链式协作

在同一个工作流中，多个智能体可以形成分析链条。每个智能体除了自己的数据源数据，还能访问**上游智能体的输出**。

引用上游输出的方式：

```
方式1：使用 {{agent:<agent_id>:output}} 变量
方式2：无数据源的智能体会自动接收所有上游输出作为上下文
```

**示例：分析链条**

```
股票分析师（数据源：实时行情+历史K线）
    ↓ 输出：技术面分析报告
行业研究员（数据源：年报财务数据）
    ↓ 输出：基本面+行业分析报告（自动接收上游分析作为上下文）
辩论（两个智能体基于上述分析进行辩论）
    ↓ 输出：辩论结论
风险评审员（基于所有分析进行风险评估）
    ↓ 输出：风险矩阵
汇总（生成最终报告）
    ↓
最终报告
```

### 5.5 智能体最佳实践

1. **明确角色定位**：每个智能体聚焦一个专业领域
2. **分析框架化**：在 System Prompt 中给出结构化分析框架
3. **数据源匹配**：确保智能体的 System Prompt 与其绑定的数据源匹配
4. **引用数据源**：在 Prompt 中使用 `{{datasource.result}}` 引用数据源输出
5. **多数据源**：一个智能体可绑定多个数据源，各数据源结果用 `【名称】` 标签分隔后合并

## 6. 工作流配置

### 6.1 工作流概念

工作流（Workflow）是一个 DAG（有向无环图），由多种节点类型组成，定义了智能体的执行顺序和数据流向。

### 6.2 节点类型

| 节点类型 | 图标 | 说明 |
|---------|------|------|
| **开始** | ▶️ | 工作流入口，定义输入参数 |
| **智能体执行** | 👤 | 执行一个绑定了数据源的智能体 |
| **并行** | 📋 | 同时执行多个智能体 |
| **辩论** | 💬 | 多个智能体进行多轮辩论 |
| **交叉验证** | ✅ | 验证多个智能体输出的一致性 |
| **风险评审** | ⚠️ | 对分析结果进行风险评估 |
| **条件分支** | 🔀 | 根据条件选择执行路径 |
| **汇总** | 📄 | 汇总所有分析结果生成最终报告 |
| **数据转换** | ⚙️ | 对变量进行转换处理 |
| **结束** | ⏹️ | 工作流出口 |

### 6.3 构建工作流

#### 步骤 1：创建

进入「工作流管理」→ 「创建工作流」，填写名称和描述。

#### 步骤 2：配置输入参数

在「全局参数」中定义工作流的输入参数，如：

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `stock_code` | string | `sh600519` | 分析标的股票代码 |
| `analysis_period` | string | `1年` | 分析周期 |
| `investment_style` | string | `长期` | 投资风格 |

这些参数会在运行工作流时由用户填写或使用默认值。

#### 步骤 3：拖拽编排节点

从左侧面板拖拽节点到画布，按分析逻辑连接：

**推荐的工作流结构：**

```
[开始] → [并行：多个智能体分析]
                    ↓
               [辩论/交叉验证]
                    ↓
               [风险评审]
                    ↓
               [汇总] → [结束]
```

**高级工作流结构（带条件分支）：**

```
[开始] → [智能体A] → [条件判断]
                        ↓
              ┌───────┴───────┐
              ▼               ▼
          [智能体B]       [智能体C]
              └───────┬───────┘
                      ▼
                 [汇总] → [结束]
```

#### 步骤 4：配置每个节点

每个节点需要根据类型进行配置：

| 节点类型 | 配置项 |
|---------|--------|
| **智能体执行** | 选择要执行的智能体 |
| **并行** | 选择需要并行执行的智能体列表 |
| **辩论** | 选择参与辩论的智能体、最大轮数、是否在达成共识时停止 |
| **交叉验证** | 选择需要验证的智能体输出 |
| **风险评审** | 配置评审提示词 |
| **条件分支** | 编写条件表达式（如 `{{branch_value}} == "深度分析"`） |
| **汇总** | 配置汇总提示词模板 |
| **数据转换** | 配置变量映射规则 |

### 6.4 辩论节点详解

辩论节点允许多个智能体进行多轮辩论，产生更深入的分析：

**配置参数：**

| 参数 | 说明 | 默认值 |
|------|------|--------|
| **参与智能体** | 选择参与辩论的智能体 | — |
| **最大轮数** | 最多辩论几轮 | `3` |
| **达成共识停止** | 所有智能体意见一致时提前结束 | `否` |

**执行流程：**

```
第1轮：每个智能体收到前置分析数据 → 输出初始观点
第2轮：每个智能体收到自己和对手的上轮观点 → 反驳/补充
第3轮：同上 → 继续辩论或达成共识
```

每轮辩论的完整上下文包括：
- 原始分析数据（持续可见）
- 自己上一轮的论点
- 对手上一轮的论点

### 6.5 工作流执行机制

```
拓扑排序 → 按依赖顺序执行节点
               ↓
        并行节点：同时执行
        串行节点：顺序执行
        条件节点：根据表达式选择分支
               ↓
        每个节点输出存入变量池
               ↓
        所有节点完成后生成报告
```

**执行配置：**

| 参数 | 说明 | 默认值 |
|------|------|--------|
| **最大并发数** | 并行执行的最大节点数 | `32` |
| **任务超时** | 整个任务最大执行时间 | `30分钟` |

## 7. 任务执行与监控

### 7.1 运行工作流

**方式一：从工作流列表页**
进入「工作流管理」→ 点击工作流的「运行」按钮 → 填写输入参数 → 确认

**方式二：从工作流编辑器**
在工作流编辑器中点击「运行」→ 填写参数 → 确认

**方式三：从任务列表页**
直接点击「创建任务」→ 选择工作流 → 填写参数 → 提交

### 7.2 输入参数

运行工作流时，需要填写工作流定义的**全局参数**。例如：

| 参数 | 值 | 说明 |
|------|-----|------|
| stock_code | `sh600519` | 贵州茅台 A 股 |
| analysis_period | `1年` | 分析过去 1 年的数据 |
| investment_style | `价值投资` | 以价值投资视角分析 |

参数值会通过以下方式影响分析：
1. 直接替换数据源 URL 中的 `{{stock_code}}` 占位符
2. 替换智能体 System Prompt 中的 `{{stock_code}}` 变量
3. 传递给数据源的 `ResolveParams` 进行参数解析

### 7.6 常见失败原因排查

通过「执行日志」标签页排查问题：

| 日志事件 | 可能原因 |
|---------|---------|
| `datasource extracted 0 chars` | 数据源返回空数据 |
| `datasource id=xxx not found` | 数据源记录已被删除 |
| `resolve datasource params failed` | 工作流输入参数未提供所需的变量 |
| `datasource execution failed` | 外部 API 不可用或返回错误 |
| `agent not found` | 智能体配置引用了不存在的智能体 |
| `context deadline exceeded` | LLM 模型超时（增大模型的超时配置） |
| `upstream timeout` | 数据源 API 超时 |
| `upstream auth failed` | API Key 错误或已过期 |

## 9. 最佳实践

### 9.1 推荐的股票分析工作流

```
[开始]
  输入参数：stock_code, analysis_period, investment_style
[智能体执行：股票分析师]
  数据源：实时行情 + 历史行情 + 财务指标
  输出：技术面+基本面分析报告
[智能体执行：行业研究员]
  数据源：年报财务数据 + 沪深行情概览 + 财务指标
  输出：行业分析报告（自动接收上游分析）
[辩论]
  参与：股票分析师 vs 行业研究员
  轮数：2轮
  输出：辩论结论
[风险评审]
  基于所有分析输出风险评估矩阵
[汇总]
  整合所有分析结果生成最终报告
[结束]
```

### 9.2 Prompt 编写技巧

1. **结构化输出**：在 Prompt 中明确要求 Markdown 格式和标题层级
2. **分析框架**：给出分析维度和框架，让 AI 有章可循
3. **角色设定**：明确的角色身份（"你是一位 XX 专家"）提升输出质量
4. **约束条件**：指明输出长度、语言、格式等要求
5. **引用数据**：要求 AI 基于数据源分析，而非其训练知识

### 9.3 多数据源策略

- **互补原则**：选择能覆盖不同分析维度的数据源组合
- **主次分明**：主要数据源提供核心数据，辅助数据源提供补充信息
- **格式统一**：多个数据源的 System Prompt 统一引用方式为 `{{datasource.result}}`

### 9.4 性能优化

- **缓存设置**：不频繁变化的数据（如年报）设置较长 TTL
- **超时配置**：根据数据量和 LLM 响应速度合理设置超时
- **并发控制**：在 Workflow 执行配置中调整最大并发数

## 10. 常见问题

### Q1：如何测试 LLM 模型是否可用？
A：在模型配置页点击「测试」按钮，如果返回正常响应说明配置正确。

### Q2：数据源返回空数据怎么办？
A：检查参数是否正确传递，URL 模板中的 `{{变量名}}` 是否与工作流输入参数名一致。在「执行日志」中查看 `allVars keys` 确认变量是否存在。

### Q3：智能体输出不理想？
A：优化 System Prompt：给出更明确的角色设定、分析框架和输出要求。确保 Prompt 中使用了 `{{datasource.result}}` 引入数据。

### Q4：辩论为空？
A：确保辩论节点前面有智能体执行节点产生分析内容。辩论需要基于前置分析进行。

### Q5：任务超时如何解决？
A：在模型配置中增大超时时间（建议 180s 以上），或在 `.env` 中调整 `ENGINE_TASK_TIMEOUT`（默认 30 分钟）。

### Q6：生成的报告日期不正确？
A：系统会自动注入 `{{current_date}}` 和 `{{report_date}}` 变量到每个 LLM 调用中。请在 System Prompt 中使用这些变量来引用当前日期。

### Q7：如何配置需要 API Key 的外部数据源？
A：选择 `api_key_header` 或 `bearer` 认证方式，在配置中填写 Header 名称和 Key 值。Key 会被加密存储。

### Q8：可以一个智能体绑定多个数据源吗？
A：可以。在编辑智能体时，绑定数据源使用多选下拉框，选择一个或多个数据源。多数据源的结果会用 `【数据源名称】` 标签分隔后合并。
